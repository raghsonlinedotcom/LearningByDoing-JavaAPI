Java Platform Module System (JPMS) Tutorial

JPMS was introduced in Java 9 to solve the "JAR Hell" problem by:

  • Enabling strong encapsulation
  • Improving maintainability and tooling
  • Allowing explicit module dependencies

A module is a self-contained unit of code with a module-info.java descriptor that defines its exports and dependencies.

module com.invoice.core {
  exports com.invoice.model;
}

Imagine a modular invoice system with three modules:

  • com.invoice.model — contains domain classes
  • com.invoice.service — business logic
  • com.invoice.app — CLI or Web entry point
