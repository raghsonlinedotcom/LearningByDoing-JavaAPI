Java Platform Module System (JPMS)

JPMS was introduced in Java 9 to provide strong encapsulation and reliable configuration of Java code.

This tutorial explains What modules are, Why they matter, How to use them, and When to apply them.

🎯 Why Use Modules?

  • ✅ Enforce encapsulation (no more public everything!)
  • ✅ Resolve dependency conflicts with explicit requires
  • ✅ Allow selective reflection via opens
  • ✅ Support plugin systems via provides/uses

JPMS solves classpath hell and enables large-scale Java application structuring.

🪄 JPMS Keywords Explained

KeywordDescription
moduleDeclares a named module
requiresImports another module
exportsMakes a package accessible to other modules
opensAllows reflection on package (needed for frameworks)
provides ... withDeclares a service implementation
usesDeclares a service consumer

🛠️ How to Compile & Run

javac --module-source-path . -d out $(find . -name "*.java")
java --module-path out -m lib.ui/com.lib.ui.LibraryApp

Ensure all modules are defined with a module-info.java file at the root.

📚 Real-World Use Cases

  • 📦 Enterprise applications with 10+ modules
  • 🔌 Plugin systems with ServiceLoader
  • 🧪 SDKs and APIs that want to expose only a small public surface
  • 🐳 Modular Docker builds and deployment (per-feature images)
