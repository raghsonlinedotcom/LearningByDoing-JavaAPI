1. ‚ùì Why Were Modules Introduced in Java?

What problems did Java have before Java 9?

  • No control over public classes ‚Äî everything was globally visible.
  • Classpath hell ‚Äî no separation of concerns or dependency safety.
  • No native plugin or service discovery mechanism.

What does JPMS solve?

  • Allows us to explicitly declare dependencies between modules.
  • Exports only what‚Äôs needed; hides everything else.
  • Supports runtime-safe service discovery using ServiceLoader.
Takeaway: JPMS brings structure and predictability to Java projects ‚Äî especially large codebases.

2. üì¶ What Is a Java Module?

A Java module is a named group of packages with a module-info.java descriptor file that explicitly declares:

  • What it requires (dependencies)
  • What it exports (public APIs)
  • What it opens (reflection)
  • What it provides and uses (services)

How do you define a module?

module com.myapp.billing {
  requires com.myapp.core;
  exports com.myapp.billing;
}
Takeaway: A module isn't just a folder ‚Äî it‚Äôs a declaration of boundaries and contracts.
